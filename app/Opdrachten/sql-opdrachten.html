Niveau 1 SELECT — alles & kolommen ophalen
01
SELECTAlle studenten ophalen
Haal alle gegevens op van alle studenten uit de tabel.
▼
📌 Uitleg Met SELECT * selecteer je alle kolommen. Met FROM geef je de tabelnaam op. Dit is de eenvoudigste SQL-query die bestaat.
SELECT * FROM tabel_naam;
SELECT * FROM studenten;
🔍 Controllevraag
Wat betekent de * in een SELECT statement?
A Selecteer alleen de eerste rij
B Selecteer alle kolommen van de tabel
C Tel het aantal rijen
D Verwijder alle gegevens
✅ Correct! De * staat voor "alle kolommen". Je kunt ook specifieke kolomnamen schrijven, zoals: SELECT voornaam, leeftijd FROM studenten;
02
SELECTSpecifieke kolommen ophalen
Haal alleen de voornaam, achternaam en klas op van alle studenten.
▼
📌 Uitleg In plaats van * kun je specifieke kolomnamen opgeven, gescheiden door een komma. Je ziet dan alleen die kolommen in je resultaat.
SELECT kolom1, kolom2, kolom3 FROM tabel;
SELECT voornaam, achternaam, klas
FROM studenten;
🔍 Controllevraag
Je wilt alleen voornaam en leeftijd zien. Welke query is correct?
A SELECT * FROM studenten WHERE voornaam, leeftijd;
B SELECT voornaam, leeftijd FROM studenten;
C SELECT voornaam + leeftijd FROM studenten;
D GET voornaam, leeftijd FROM studenten;
✅ Goed! Kolomnamen worden gescheiden door komma's. GET bestaat niet in SQL — het juiste keyword is SELECT.
03
SELECTResultaat sorteren met ORDER BY
Haal alle studenten op, gesorteerd op achternaam van A naar Z.
▼
📌 Uitleg ORDER BY kolomnaam ASC sorteert oplopend (A→Z of laag→hoog). DESC sorteert aflopend (Z→A). Zonder dit keyword is de volgorde willekeurig.
Voeg toe aan het einde: ORDER BY achternaam ASC
SELECT *
FROM studenten
ORDER BY achternaam ASC;
🔍 Controllevraag
Wat is het verschil tussen ASC en DESC bij ORDER BY?
A ASC sorteert op tekst, DESC op getallen
B Er is geen verschil
C ASC = oplopend (A→Z), DESC = aflopend (Z→A)
D ASC = aflopend, DESC = oplopend
✅ Correct! ASC = ascending (oplopend) en DESC = descending (aflopend). Als je niets opgeeft, is ASC de standaard in MySQL.
Niveau 2 SELECT — Filteren met WHERE
04
SELECTFilteren op tekst
Haal alle studenten op die in klas 1A zitten.
▼
📌 Uitleg Met WHERE filter je rijen op een voorwaarde. Tekst moet je tussen aanhalingstekens zetten: '1A'. Getallen schrijf je zonder aanhalingstekens: leeftijd = 16.
WHERE klas = '1A'
SELECT *
FROM studenten
WHERE klas = '1A';
🔍 Controllevraag
Waarom staat '1A' tussen aanhalingstekens, maar een getal zoals 16 niet?
A Aanhalingstekens zijn altijd optioneel
B Tekst (strings) krijgen aanhalingstekens, getallen niet
C Omdat '1A' met een letter eindigt
D Kolomnamen krijgen altijd aanhalingstekens
✅ Juist! Tekst-waarden (strings) zet je altijd tussen aanhalingstekens. Getallen schrijf je direct: WHERE leeftijd = 16. Schrijf je toch '16', dan werkt het in MySQL toevallig ook — maar het is niet de correcte schrijfwijze.
05
SELECTFilteren met vergelijkingsoperatoren
Haal alle studenten op die 16 jaar of ouder zijn, gesorteerd op leeftijd van laag naar hoog.
▼
📌 Uitleg Vergelijkingsoperatoren: = (gelijk), > (groter), >= (groter of gelijk), < (kleiner), <= (kleiner of gelijk), != of <> (niet gelijk).
WHERE leeftijd >= 16 daarna ORDER BY leeftijd ASC
SELECT *
FROM studenten
WHERE leeftijd >= 16
ORDER BY leeftijd ASC;
🔍 Controllevraag
Welke operator gebruik je om studenten te selecteren die NIET 15 jaar oud zijn?
A leeftijd == 15
B leeftijd != 15 of leeftijd <> 15
C NOT leeftijd 15
D leeftijd =! 15
✅ Juist! Zowel != als <> betekenen "niet gelijk aan". In SQL gebruik je één = voor gelijkheid — == bestaat niet in SQL.
06
SELECTCombineren met AND & OR
Haal alle studenten op die in klas 1B zitten én 16 jaar oud zijn.
▼
📌 Uitleg AND = beide voorwaarden moeten waar zijn. OR = minstens één voorwaarde moet waar zijn. Combineer ze met haakjes om de volgorde duidelijk te maken: (klas = '1A' OR klas = '1B') AND leeftijd = 16.
WHERE klas = '1B' AND leeftijd = 16
SELECT *
FROM studenten
WHERE klas = '1B' AND leeftijd = 16;
🔍 Controllevraag
Je wilt studenten uit klas 1A óf klas 1C ophalen. Welke query werkt correct?
A WHERE klas = '1A' AND klas = '1C'
B WHERE klas = '1A' OR klas = '1C'
C WHERE klas = '1A', '1C'
D WHERE klas = '1A' PLUS klas = '1C'
✅ Correct! Een student kan niet tegelijk in 1A én 1C zitten, dus AND geeft 0 resultaten. Met OR is één van de twee voldoende. PLUS bestaat niet in SQL.
Niveau 3 SELECT — BETWEEN, IN & LIMIT
07
SELECTBereik filteren met BETWEEN
Haal alle studenten op die 15 of 16 jaar oud zijn. Gebruik BETWEEN.
▼
📌 Uitleg BETWEEN x AND y selecteert alle waarden inclusief de grenzen. Het is een kortere schrijfwijze voor leeftijd >= 15 AND leeftijd <= 16. Werkt ook op tekst en datums.
WHERE leeftijd BETWEEN 15 AND 16
SELECT *
FROM studenten
WHERE leeftijd BETWEEN 15 AND 16;
🔍 Controllevraag
Is BETWEEN 15 AND 16 hetzelfde als leeftijd >= 15 AND leeftijd <= 16?
A Ja, BETWEEN is inclusief de grenzen (15 en 16 doen mee)
B Nee, BETWEEN sluit de grenzen uit
C Alleen bij getallen zijn ze hetzelfde
D BETWEEN werkt alleen met datums
✅ Juist! BETWEEN is altijd inclusief in MySQL. BETWEEN 15 AND 16 geeft precies hetzelfde resultaat als >= 15 AND <= 16. Handig om te weten: BETWEEN werkt ook op tekst (alfabetisch) en datum-kolommen.
08
SELECTLijst filteren met IN
Haal alle studenten op die in Amsterdam, Rotterdam of Utrecht wonen. Gebruik IN.
▼
📌 Uitleg IN (waarde1, waarde2, ...) is een compacte schrijfwijze voor meerdere OR-voorwaarden op dezelfde kolom. In plaats van drie losse OR-regels schrijf je één overzichtelijke lijst.
WHERE woonplaats IN ('Amsterdam', 'Rotterdam', 'Utrecht')
SELECT *
FROM studenten
WHERE woonplaats IN ('Amsterdam', 'Rotterdam', 'Utrecht')
ORDER BY woonplaats ASC;
🔍 Controllevraag
Welke query is gelijkwaardig aan WHERE klas IN ('1A', '1B')?
A WHERE klas = '1A' AND klas = '1B'
B WHERE klas = '1A' OR klas = '1B'
C WHERE klas BETWEEN '1A' AND '1B'
D WHERE klas = ('1A', '1B')
✅ Helemaal goed! IN is een nettere schrijfwijze voor meerdere OR-condities op dezelfde kolom. Hoe langer de lijst, hoe meer voordeel IN heeft.
09
SELECTResultaten beperken met LIMIT
Haal de eerste 10 studenten op, gesorteerd op id van laag naar hoog.
▼
📌 Uitleg LIMIT n beperkt het aantal teruggegeven rijen. Handig bij grote tabellen. Met LIMIT n OFFSET m sla je de eerste m rijen over — dit gebruik je voor paginering (pagina 1, 2, 3…).
ORDER BY id ASC daarna: LIMIT 10
SELECT *
FROM studenten
ORDER BY id ASC
LIMIT 10;
🔍 Controllevraag
Wat geeft LIMIT 10 OFFSET 20 terug?
A De eerste 20 rijen
B Rijen 10 tot en met 20
C 10 rijen, beginnend ná de 20ste rij (rijen 21 t/m 30)
D De laatste 10 rijen van de tabel
✅ Precies! OFFSET slaat een aantal rijen over. Pagina 1 = LIMIT 10 OFFSET 0, pagina 2 = LIMIT 10 OFFSET 10, enz. Dit is het standaard paginerings-patroon.
Niveau 4 UPDATE — Gegevens aanpassen
⚠️
Let op bij UPDATE!

Vergeet je de WHERE clausule, dan worden alle rijen in de tabel gewijzigd. Controleer altijd eerst met een SELECT welke rijen je gaat aanpassen.

10
UPDATEEén waarde aanpassen
Student met id 3 is verhuisd naar Groningen. Pas de woonplaats aan.
▼
📌 Uitleg Structuur: UPDATE tabel SET kolom = waarde WHERE voorwaarde;. De WHERE zorgt dat je alleen de bedoelde rij aanpast. Gebruik altijd de id (primaire sleutel) als je één specifieke rij wilt aanpassen.
UPDATE studenten SET woonplaats = 'Groningen' WHERE id = 3;
UPDATE studenten
SET woonplaats = 'Groningen'
WHERE id = 3;
🔍 Controllevraag
Wat gebeurt er als je UPDATE studenten SET woonplaats = 'Groningen' uitvoert zonder WHERE?
A Er gebeurt niets, SQL vraagt om bevestiging
B Alleen de eerste rij wordt aangepast
C Alle 100 studenten krijgen woonplaats 'Groningen'
D SQL geeft een foutmelding want WHERE is verplicht
✅ Correct! Zonder WHERE past UPDATE elke rij aan — een veelgemaakte fout. Tip: voer altijd eerst SELECT * FROM studenten WHERE id = 3; uit om te controleren welke rij je gaat wijzigen.
11
UPDATEMeerdere kolommen tegelijk aanpassen
Student met id 7 heeft verjaardag gehad (leeftijd wordt 18) en is overgezet naar klas 1C. Pas beide velden in één query aan.
▼
📌 Uitleg Je kunt meerdere kolommen tegelijk aanpassen door ze na SET met komma's te scheiden: SET kolom1 = waarde1, kolom2 = waarde2. Efficiënter dan twee losse UPDATE-queries.
UPDATE studenten SET leeftijd = 18, klas = '1C' WHERE id = 7;
UPDATE studenten
SET leeftijd = 18, klas = '1C'
WHERE id = 7;
🔍 Controllevraag
Alle studenten in klas 1A worden verplaatst naar 1B. Welke query is correct?
A UPDATE studenten SET klas WHERE klas = '1A' = '1B';
B UPDATE studenten WHERE klas = '1A' SET klas = '1B';
C UPDATE studenten SET klas = '1B' WHERE klas = '1A';
D CHANGE studenten SET klas = '1B' WHERE klas = '1A';
✅ Correct! De volgorde is altijd: UPDATE tabel SET kolom = waarde WHERE voorwaarde. Hiermee worden alle rijen aangepast waar klas gelijk is aan '1A'. CHANGE bestaat niet in SQL.
Niveau 5 DELETE — Gegevens verwijderen
🚨
Pas op: DELETE is onomkeerbaar!

Verwijderde rijen zijn permanent weg — er is geen prullenbak in SQL. Gebruik altijd WHERE en voer van tevoren een SELECT uit met dezelfde WHERE om te controleren wat er verwijderd gaat worden.

12
DELETEEén rij verwijderen
Student met id 5 heeft zich uitgeschreven. Verwijder deze student uit de database.
▼
📌 Uitleg Structuur: DELETE FROM tabel WHERE voorwaarde;. Je schrijft geen kolomnamen — DELETE verwijdert altijd hele rijen. Gebruik de primaire sleutel (id) om precies één rij te verwijderen.
DELETE FROM studenten WHERE id = 5;
DELETE FROM studenten
WHERE id = 5;
🔍 Controllevraag
Wat doet DELETE FROM studenten; (zonder WHERE)?
A De tabel zelf wordt verwijderd
B Alleen de eerste rij wordt verwijderd
C Alle 100 rijen worden verwijderd; de lege tabel blijft bestaan
D SQL geeft een foutmelding want WHERE is verplicht
✅ Correct! Zonder WHERE verwijdert DELETE alle rijen, maar de tabelstructuur blijft intact. Wil je ook de tabel zelf verwijderen? Dan gebruik je DROP TABLE studenten;.
13
DELETEMeerdere rijen verwijderen
Klas 1C wordt opgeheven. Verwijder alle studenten die in klas 1C zitten.
▼
📌 Uitleg DELETE verwijdert alle rijen die aan de WHERE-voorwaarde voldoen. Je kunt dezelfde filter-technieken gebruiken als bij SELECT: =, IN, AND, enz. Controleer van tevoren met: SELECT * FROM studenten WHERE klas = '1C';
DELETE FROM studenten WHERE klas = '1C';
DELETE FROM studenten
WHERE klas = '1C';
🔍 Controllevraag
Wat is de beste werkwijze vóór het uitvoeren van een DELETE met WHERE?
A Gewoon uitvoeren, DELETE vraagt automatisch om bevestiging
B Eerst een SELECT met dezelfde WHERE uitvoeren om te zien welke rijen verwijderd worden
C Niets, DELETE is altijd veilig met WHERE
D De tabel vergrendelen met LOCK TABLE eerst
✅ Uitstekend! Voer altijd eerst SELECT * FROM studenten WHERE klas = '1C'; uit. Zo zie je precies hoeveel en welke rijen worden verwijderd — en kun je een onbedoelde fout voorkomen. Bij productiedatabases maak je ook altijd een backup.
